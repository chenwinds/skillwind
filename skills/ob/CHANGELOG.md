# OB 技能 - 变更日志

## v1.2.0 (2026-03-13)

### 评估工具重构

#### 新增功能
1. **三种运行模式**
   - `--mode eval` - 评估单次输出质量
   - `--mode add` - 收录高分输出到测试用例
   - `--mode batch` - 批量运行所有测试用例

2. **自动收录功能**
   - 评估得分为满分时，提示收录
   - 收录时自动分析输出特征，生成评估标准
   - 自动添加到 `evals.json`

3. **文字报告输出**
   - 删除 HTML 查看器，只保留文字报告
   - 评估结果清晰展示：得分、通过状态、详细评分
   - 低分时显示需要改进的项目
   - 高分时显示收录建议

4. **自迭代工作流**
   ```
   运行技能 → 评估输出 → 高分收录 → 扩充测试集 → 批量验证
   ```

#### 评估标准（满分 15 分）

| 项目 | 分值 | 检查内容 |
|------|------|----------|
| 有标点符号 | 2 | 包含，,.，。？！等标点 |
| 有合理分段 | 1 | 至少 2 个段落 |
| 无多余开场白 | 2 | 没有"好的""以下是"等 |
| 无语气词 | 3 | 没有呃哦啊嗯嘛哎诶呀哇 |
| 无填充词 | 2 | 没有那个然后就是其实 basically |
| 使用 Markdown 标题 | 2 | 有#标题 |
| 使用列表 | 1 | 有列表格式 |

通过线：80% = 12 分

## v1.1.0 (2026-03-13)

### 主要改进

#### 1. 删除冗余文件
- 删除了 `ob.skill` ZIP 压缩包，该文件与 `SKILL.md` 内容重复
- 减少了文件冗余，保持项目整洁

#### 2. 重构文件结构
- 创建 `references/` 目录
- 将 `examples.md` 移至 `references/examples.md`
- 修正了 `SKILL.md` 中的引用路径
- 创建 `scripts/` 目录存放工具脚本

**新文件结构：**
```
skills/ob/
├── SKILL.md                    # 技能定义
├── evals.json                  # 评估配置
├── CHANGELOG.md                # 变更日志
├── references/
│   └── examples.md             # 输入输出示例
└── scripts/
    └── run_eval.py             # 自动化评估脚本
```

#### 3. 量化评估标准
- 重写 `evals.json`，添加结构化评估标准
- 每个评估项包含：
  - `criterion`: 评估标准描述
  - `weight`: 权重分值
  - `type`: 评估类型（binary 等）
- 添加评分指南：得分率 >= 80% 为通过

#### 4. 边界情况处理
在 `SKILL.md` 中新增"边界情况处理"章节，包含：

| 边界情况 | 处理方式 |
|----------|----------|
| 语音识别错误 | 根据上下文修正同音字、近音词 |
| 多语言混合 | 保留英文和专有名词，正常添加中文标点 |
| 方言和口语表达 | 保留特色表达，仅添加标点 |
| 不完整句子 | 根据语义补充或保持不完整 |
| 专业术语 | 保留原文格式 |
| 数字和单位 | 保持原有格式 |

新增 5 个边界情况示例：
- 多语言混合
- 方言和口语表达
- 不完整句子和重复
- 访谈记录
- 带数字和专业术语的内容

#### 5. 优化触发条件
- 在 frontmatter 中添加 `trigger_keywords` 字段
- 扩展触发关键词列表：
  - 整理口述
  - 语音转文字
  - 会议记录
  - 访谈记录
  - 语音笔记
  - 口语化文字
  - 没有标点
  - 录音整理
- 在 `SKILL.md` 中添加详细的"触发条件"章节
- 区分"内容特征"和"关键词触发"两类触发方式

### 其他改进
- 使用表格格式化展示标点符号和 Markdown 语法
- 更新 `registry.json` 至 v1.1.0
- 添加变更同步历史记录
