# OB 技能评估工具使用说明

## 环境要求

- Python 3.8+
- 本项目使用 Miniconda Python: `D:\ai\miniconda\python.exe`

## 三种使用模式

### 模式 1: 评估单次输出质量

**用途**: 检查技能输出质量，得分 13/13 为满分

**命令**:
```bash
python run_eval.py --mode eval --prompt "输入内容" --output "技能输出"
```

**示例**:
```bash
/d/ai/miniconda/python.exe run_eval.py --mode eval ^
  --prompt "嗯今天我们开了一个会" ^
  --output "# 会议记录\n\n今天我们开了一个产品评审会。\n\n1. 第一项\n2. 第二项"
```

**输出**:
```

============================================================

得分：13/13 (100.0%)
状态：[OK] 通过（通过线：80%）
```

### 模式 2: 收录高分输出到测试用例

**用途**: 当评估得分为满分 (13/13) 时，收录为新的测试用例

**命令**:
```bash
python run_eval.py --mode add --prompt "输入" --output "输出" --name "用例名"
```

**示例**:
```bash
/d/ai/miniconda/python.exe run_eval.py --mode add ^
  --prompt "嗯今天我们开了一个会" ^
  --output "# 会议记录\n\n今天我们开了一个产品评审会。" ^
  --name "会议记录测试"
```

**输出**:
```
[评估结果同上]

[OK] 已添加到测试用例 #4: 会议记录测试
```

## 快速启动脚本

### Windows (eval.bat)
```cmd
eval.bat eval "输入" "输出"
eval.bat add "输入" "输出" "名称"
eval.bat batch
```

### Linux/Mac (eval.sh)
```bash
./eval.sh eval "输入" "输出"
./eval.sh add "输入" "输出" "名称"
./eval.sh batch
```

**注意**: 批处理文件 (`eval.bat`) 不支持多行输出。如需评估多行内容，请直接使用 Python 命令。

## 评估标准（满分 13 分）

| 项目 | 分值 | 检查内容 |
|------|------|----------|
| 有标点符号 | 2 | 包含，,.，。？！等标点 |
| 有合理分段 | 1 | 至少 2 个段落 |
| 无多余开场白 | 2 | 没有"好的""以下是"等 |
| 无语气词 | 3 | 没有呃哦啊嗯嘛哎诶呀哇 |
| 无填充词 | 2 | 没有那个然后就是其实 basically |
| 使用 Markdown 标题 | 2 | 有#标题 |
| 使用列表 | 1 | 有列表格式 |

**通过线**: 10 分 (80%)
**满分**: 13 分（可收录为测试用例）

## 完整工作流

### 迭代改进流程

```
1. 使用 OB 技能整理内容 → 获得输出
        ↓
2. 评估输出 → python run_eval.py --mode eval
        ↓
3. 查看结果
   ├─ 满分 (13/13) → 收录到测试用例 (--mode add)
   └─ 低于 10 分 → 修改技能 → 重新测试
        ↓
4. 定期批量测试 → python run_eval.py --mode batch
```

### 自迭代循环

```python
# 伪代码：技能自动评估与改进
output = run_ob_skill(prompt)
score, total = evaluate(output)

if score == total:
    # 满分，收录为测试用例
    add_to_evals(prompt, output)
    print("评估通过，已收录")
elif score >= total * 0.8:
    # 通过但非满分
    print("评估通过")
else:
    # 未通过，重新尝试
    print("评估未通过，重新调整...")
    output = retry_with_refinement(prompt)
```

## 常见问题

**Q: 为什么命令行显示中文乱码？**
A: 运行前执行 `chcp 65001` 切换到 UTF-8 编码。

**Q: 如何修改通过线？**
A: 编辑 `run_eval.py`，修改 `PASS_THRESHOLD = 0.8` 的值。

**Q: 如何查看已有的测试用例？**
A: 打开 `evals.json` 文件查看所有测试用例。
