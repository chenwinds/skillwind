# OB to Article - 口述转完整文章技能

这是一个组合技能，整合了两个独立的技能来完成从口述内容到完整配图文章的端到端处理。

## 什么时候使用这个技能

当用户提供以下内容时，这个技能会被触发：

- Word 文档（.docx 文件）- 自动检测并转换为 Markdown
- 语音识别/语音转文字的原始输出
- 大段没有标点或标点混乱的口述内容
- 访谈记录、会议记录、对话记录
- 语音笔记、口述日记
- 用户明确请求将口述内容整理成文章并配图

## 工作流程

这个技能按顺序执行四个阶段：

### 阶段 0：检测并转换 Word 文档

首先，检查用户输入是否包含 Word 文档（.docx 文件）：

1. **检测文件类型** - 判断输入是否为 .docx 文件
2. **调用 docx 技能** - 如果是 Word 文档，调用 `docx` 技能将其转换为 Markdown 格式
3. **提取文本内容** - 从转换后的 Markdown 中获取文本内容

**关键约束**：
- 仅当检测到 .docx 文件时才执行此阶段
- 如果输入不是 Word 文档，直接跳过此阶段进入阶段 1
- 转换后保留原文档的结构（标题、段落、列表等）

### 阶段 1：调用 ob 技能整理口述内容

首先，调用 `ob` 技能将口语化内容转换为结构化文章：

1. **添加标点符号** - 根据语义添加合适的标点
2. **删除语气词** - 去除呃、哦、啊、嗯等冗余语气词
3. **文章分段** - 根据逻辑分成合适的段落
4. **Markdown 格式化** - 使用标题、列表等格式化
5. **有序列表转换** - 将枚举转换为有序列表

**关键约束**：
- 只删除纯粹语气词，保留所有实质性内容
- 保持原文的核心表达和用词风格
- 直接输出整理后的内容，不要解释说明

### 阶段 2：调用 article-image-matcher 技能为文章配图

在获得整理后的文章后，调用 `article-image-matcher` 技能为文章匹配图片：

1. **分析文章内容** - 理解主题、关键词、情感基调
2. **提取搜索关键词** - 5-10 个中英文关键词
3. **搜索图片** - 从 Pexels、Pixabay、Unsplash 搜索横版图片
4. **选择 3-5 张图片** - 1 张封面 + 2-4 张章节配图，必须为横版
5. **嵌入 Markdown** - 在合适位置插入图片

**关键约束**：
- 只选择横版图片（宽度 > 高度）
- 图片必须与相邻内容相关
- 封面图放在标题后，章节图放在章节开头

### 阶段 3：自动保存 Markdown 文件

完成配图后，自动将最终文章保存到本地文件系统：

1. **生成文件名** - 根据输入来源自动生成文件名：
   - 如果输入是 Word 文档：使用原名 + `_整理后` 作为文件名
   - 如果输入是口述文本：使用当前日期时间作为文件名
2. **确定保存路径**：
   - 如果输入是文件：保存到同一目录下
   - 如果输入是纯文本：保存到桌面或当前工作目录
3. **写入文件** - 使用 Write 工具将完整的 Markdown 内容保存到文件
4. **告知用户** - 输出保存后的文件路径

**关键约束**：
- 必须执行此阶段，确保输出被持久化保存
- 文件名使用 UTF-8 编码，避免乱码
- 保存后明确告知用户文件路径

## 输出格式

最终输出应包含：
1. 整理后的结构化 Markdown 文章（已保存到本地文件）
2. 内嵌的 3-5 张横版配图
3. 保存后的文件路径
4. 简要说明关键词、图片来源和放置位置

## 注意事项

- 如果用户提供 Word 文档，先转换为 Markdown 再处理
- 如果用户提供口述内容，先整理再配图
- 如果用户提供已整理的文章，跳过阶段 1 直接配图
- 图片搜索需要配置 API 密钥（Pixabay 必需，Pexels 推荐）
- **阶段 3 必须执行** - 最终文章必须自动保存到本地文件系统
